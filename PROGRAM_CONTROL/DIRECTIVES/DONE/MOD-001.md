## Context

The Canonical Corpus (Chapter 7, Section 6.1) defines the incident lifecycle:
`OPEN -> UNDER_REVIEW -> ACTIONED -> CLOSED`. Every transition must be logged with
severity (SEV1/SEV2/SEV3), assigned role, reason code, and evidence hash.
No silent transitions. No informal resolution.

## Tasks

1. Create `services/core-api/src/safety/incident.service.ts` — full incident lifecycle state machine
2. Add `IncidentService` to `SafetyModule` providers and exports
3. Add NATS topic `INCIDENT_TRANSITION: 'moderation.incident.transition'` to topics registry

## Validation

- `transition()` throws on invalid transitions (e.g., OPEN -> ACTIONED, CLOSED -> OPEN)
- `transition()` succeeds on valid transitions with reason code
- `transition()` stamps `closed_at_utc` when transitioning to CLOSED
- `getRequiredRole()` returns correct roles per severity
- `computeSlaDeadline()` returns 48h for NCII, null for non-SLA categories
- `isSlaBreach()` returns correct results based on deadline and status
- NATS published on every valid transition
- `npx tsc --noEmit` zero new errors
