- `DigitalVaultService.deliverToVault(userId, productId, orderId, orderStatus): DeliveryResult`
  - **Gate**: Blocks delivery and emits `FAILED_DELIVERY` unless `orderStatus === 'PAID'`
    (Doctrine §2829–2830).
  - On success, creates a `VaultItem` with:
    - `vault_item_id` — `randomUUID()` (Doctrine §2800).
    - `order_id` — immutable link to the originating order for audit replayability
      (Doctrine §2799).
    - `delivered_at` — America/Toronto timestamp (Doctrine §2800).
    - `watermark_fingerprint` — from `WatermarkUtility` (Doctrine §2800).
    - `rule_applied_id: 'Doctrine §2789'` — "All Sales Final" (Doctrine §2789).
  - Emits `DELIVERED_TO_VAULT` audit event carrying `rule_applied_id`, `order_id`,
    `vault_item_id`, and `platform_time` (Doctrine §2830).
- `DigitalVaultService.trackDownload(vaultItemId): DownloadEvent | null`
  - Increments `download_count` on the vault item.
  - Logs a `DOWNLOAD_TRACKED` event with `platform_time` in America/Toronto
    (Doctrine §2800).
  - Returns `null` for unknown IDs; emits a `VAULT_ITEM_NOT_FOUND` warning.
